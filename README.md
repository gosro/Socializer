# Socializer

Персональный полу-автономный AI-агент для общения в Telegram. Логинится **как ты**
(юзербот на MTProto), от твоего имени ведёт переписку с **выбранными тобой** контактами
в твоём стиле, помнит контекст в markdown-файлах и отвечает человекоподобно. Все важные
решения (приглашения, чувствительные темы, черновики) приходят тебе на апрув в отдельный
управляющий Telegram-бот.

> Дизайн и план: [`docs/superpowers/specs/2026-07-19-socializer-design.md`](docs/superpowers/specs/2026-07-19-socializer-design.md),
> [`docs/superpowers/plans/2026-07-19-socializer-mvp.md`](docs/superpowers/plans/2026-07-19-socializer-mvp.md).

## ⚠️ Прочитай перед запуском

- **Риск бана.** Автоматизация пользовательского аккаунта нарушает ToS Telegram —
  аккаунт можно потерять. Проект снижает риск (человекоподобные задержки, дневные лимиты,
  ночной сон, «прогрев», kill switch), но не убирает его полностью.
- **Стратегия «черновики-first».** На старте держи все контакты в режиме `draft`: агент
  готовит ответ, но НЕ отправляет — присылает тебе на апрув. Пока нет автоотправки, риск
  близок к нулю. Переводи контакт в `auto` только когда убедился, что черновики звучат
  как ты.
- **Только белый список.** Агент работает исключительно с контактами из `data/contacts.yaml`.
  Он не ходит по чужим чатам и не собирает чужую переписку.
- **Без манипуляций.** Агент общается честно, как обычный человек (может сказать «не знаю»),
  не выполняет задания собеседника и защищён от prompt injection.
- **Приглашения и встречи** всегда требуют твоего апрува — агент никогда не соглашается на
  планы в реальной жизни сам.

## Что нужно

- Python 3.11+
- Аккаунт Telegram + `api_id` / `api_hash` с https://my.telegram.org/apps
- Бот от [@BotFather](https://t.me/BotFather) (для управляющего бота)
- Твой числовой user id (узнать у [@userinfobot](https://t.me/userinfobot))
- Ключ LLM-провайдера (по умолчанию — [OpenRouter](https://openrouter.ai), модель
  `deepseek/deepseek-chat`)

## Установка

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

## Настройка

**1. Секреты.** Скопируй шаблон и заполни:

```bash
cp .env.example .env
# отредактируй .env: TELEGRAM_API_ID, TELEGRAM_API_HASH,
# CONTROL_BOT_TOKEN, OWNER_USER_ID, OPENROUTER_API_KEY
```

**2. Твой стиль.** Собери `personality.md` из твоих сообщений (интерактивно выберешь чаты):

```bash
python scripts/build_personality.py
```

Открой `data/personality.md` и поправь при желании — это твой стиль письма.

**3. Контакты.** Скопируй пример и заполни своими:

```bash
mkdir -p data
cp contacts.example.yaml data/contacts.yaml
# впиши свои контакты; на старте ВСЕМ поставь mode: draft
```

**4. (Опционально) Настройки** в `config.yaml`: модель LLM, лимиты сообщений, тайминги,
часы ночного сна.

## Запуск

```bash
PYTHONPATH=src python -m socializer.main
```

При первом запуске Telethon попросит войти (телефон + код). Оба клиента (юзербот и
управляющий бот) поднимутся в одном процессе.

**Управление — через управляющий бот** (пиши ему командами):

| Команда | Что делает |
|---------|-----------|
| `/status` | что агент делает сейчас, сколько апрувов в очереди |
| `/pause` / `/resume` | временно выключить / включить |
| `/kill` | **стоп** всей отправки (kill switch) |
| `/contacts` | список контактов |
| `/pending` | ожидающие апрувы |

Апрувы приходят кнопками: для `draft` — **Отправить / Править / Пропустить**;
для важного (приглашения, чувствительное) — **Ответить так / Написать своё / Игнорировать**.

## Рекомендуемый порядок обкатки

1. Все контакты в `mode: draft`.
2. Напиши своему юзербот-аккаунту с другого аккаунта → убедись, что черновик пришёл в бот
   и собеседнику **ничего** не ушло без твоего апрува.
3. Итерируй `personality.md`, пока черновики не будут звучать как ты.
4. Переведи **один** проверенный контакт в `mode: auto`, понаблюдай.
5. Дальше расширяй по мере доверия.

## Тесты

```bash
PYTHONPATH=src pytest -q
```

Все тесты работают на моках — реальные Telegram/LLM API не дёргаются.

## Структура

```
src/socializer/
├── telegram/      # юзербот: клиент, listener (whitelist), human-like sender
├── control_bot/   # управляющий бот: команды, апрувы, кнопки
├── approval/      # очередь ожидающих апрувов
├── brain/         # context builder, responder, safety gate, orchestrator, proactive
├── llm/           # провайдер-агностик (OpenRouter/OpenAI/Kimi, Anthropic)
├── memory/        # personality.md + memory/<контакт>.md
├── safety/        # тайминг, лимиты, kill switch
├── config.py      # загрузка config.yaml + .env
└── main.py        # сборка всего в один asyncio-процесс
scripts/
└── build_personality.py   # интерактивный сбор твоего стиля
data/               # приватное (gitignored): сессия, ключи, память, контакты
```

## Приватность

Вся приватная информация — сессия Telegram, ключи, `personality.md`, память по контактам,
контакты, очередь апрувов — лежит в `data/` и **не попадает в git** (`.gitignore`).
Дословная переписка собеседников не хранится — только редактируемые markdown-сводки, которые
ты можешь открыть, поправить или удалить.
